Visual Structure & Layout

Foundation for perception, hierarchy, and navigation — how the page is structured spatially.

Grid System

The layout uses a 12-column CSS grid with a maximum content width of 1200px and consistent horizontal padding — 32px on desktop, 16px on mobile. This is the same grid system used across professional web design (Bootstrap, Material Design).

Columns allow flexible content arrangements: full-width sections for heroes and bands; 2-column for content+sidebar; 3-column for card grids; single column on mobile.

12-column grid · max-width 1200px · gap 24px

Research link → DR6, F2

A consistent grid creates predictable spatial relationships, reducing cognitive load for older users (Sunita) who found existing platforms "too many things." Predictable structure = learnability.

Spacing & Rhythm

All spacing uses a base-4 scale (4px, 8px, 16px, 24px, 32px, 48px, 64px, 96px, 128px). This creates visual rhythm — consistent distances between elements signal grouping and hierarchy without needing borders or background colours.

Wireframe: Homepage Layout

The homepage uses a full-bleed hero, followed by alternating full-width and contained sections. This creates visual breathing room and prevents the "wall of content" that frustrated research participant Sunita (F2).

Navigation Structure

Navigation is persistent (sticky top bar on desktop) and collapses to a hamburger drawer on mobile screens below 768px. Maximum navigation depth is 2 levels — a primary nav and one sub-level for the Design System pages.

Active page state uses both colour (terracotta underline) and underline — not colour alone — satisfying WCAG 1.4.1 (Use of Colour). A skip-to-content link is the first focusable element on every page (WCAG 2.4.1).
